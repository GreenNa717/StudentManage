# 当前状态说明

本文档只描述当前仓库代码已经实现的内容，不描述规划中但尚未落地的能力。

## 项目阶段

当前项目属于“可运行、可演示、可继续扩展”的阶段。

已经具备：

- 完整的前后端工程结构
- 可初始化的数据库脚本
- 基础教务数据管理能力
- 登录认证与角色区分
- 部分关键数据范围控制
- 基础测试与文档

还不具备：

- 完整用户后台
- 更细的菜单/按钮权限体系
- 导入导出和批量处理
- 完整审计、日志、监控、部署体系
- 覆盖全面的自动化测试

## 后端现状

### 已实现模块

- 认证模块
  - `POST /api/auth/login`
  - `GET /api/auth/info`
  - `PUT /api/auth/password`
- 仪表盘模块
  - 概览统计
  - 院系学生分布
  - 成绩分布
- 基础资料模块
  - 院系 `departments`
  - 专业 `majors`
  - 班级 `classes`
  - 教师 `teachers`
  - 学生 `students`
  - 课程 `courses`
  - 成绩 `scores`

### 已实现能力

- Spring Security + JWT 鉴权
- 统一响应结构
- 全局异常处理
- 分页查询
- 逻辑上的角色权限控制
- 成绩、课程、学生三类关键数据的数据范围控制

### 业务校验现状

- 院系删除前校验是否存在专业或教师
- 专业删除前校验是否存在班级
- 班级删除前校验是否存在学生
- 教师删除前校验是否存在课程
- 课程删除前校验是否存在成绩
- 学生学号唯一
- 教师工号唯一
- 课程号唯一
- 成绩范围 `0-100`
- 同一学生 + 同一课程 + 同一学期的成绩唯一

## 前端现状

### 已实现页面

- 登录页
- 首页仪表盘
- 个人资料页
- 院系管理
- 专业管理
- 班级管理
- 教师管理
- 学生管理
- 课程管理
- 成绩管理

### 已实现交互

- 登录后根据角色生成菜单
- 请求自动携带 token
- 未登录自动跳回登录页
- 通用 CRUD 页面组件
- 仪表盘图表展示
- 主题切换与暗色模式切换
- 个人资料查看与修改密码

### 当前角色视图

- 管理员
  - 可见：院系、专业、班级、教师、学生、课程、成绩、仪表盘、个人资料
  - 可操作：大多数基础资料和成绩的增删改查
- 教师
  - 可见：学生信息、课程信息、成绩管理、仪表盘、个人资料
  - 可操作：只能维护本人授课课程的成绩
- 学生
  - 可见：我的信息、我的成绩、仪表盘、个人资料
  - 可操作：只能查看本人信息并修改本人密码

## 测试现状

当前已落地的自动化验证包括：

- 后端单元/集成测试可运行
- 新增了关键权限测试：
  - 学生不能查看其他学生
  - 学生不能查看其他学生成绩
  - 教师不能查看其他教师课程
  - 学生不能查看无关课程
  - 教师不能维护其他教师课程成绩
- 前端可以通过 `lint` 和生产构建

## 主要缺口

- 缺少系统用户管理、重置密码、账号启停等后台能力
- 缺少导入导出和批量业务处理
- 缺少更完整的错误页和异常导航体验
- 缺少接口文档生成、CI、部署脚本和环境分层配置
- 前端构建体积偏大，首页图表相关 chunk 仍有体积警告

## 适合下一步优先补的方向

1. 用户管理与账号生命周期
2. 前端 403/404 与深链路由体验
3. 成绩导入导出
4. 更完整的接口测试与前端 E2E 测试
5. 部署脚本与环境配置模板
