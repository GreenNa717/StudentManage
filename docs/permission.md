# 权限说明

## 角色定义

- `ADMIN`：系统管理员，拥有全部数据管理权限。
- `TEACHER`：教师账号，可查看基础资料，并维护本人授课课程的成绩。
- `STUDENT`：学生账号，仅可查看本人信息、本人课程和本人成绩。

## 页面级权限

| 页面 | ADMIN | TEACHER | STUDENT |
|---|---|---|---|
| 仪表盘 | 全量数据 | 关联数据 | 本人相关数据 |
| 学生页 | 查看/新增/修改/删除 | 仅查看 | 仅查看本人 |
| 课程页 | 查看/新增/修改/删除 | 仅查看本人授课课程 | 仅查看本人相关课程 |
| 成绩页 | 查看/新增/修改/删除 | 查看/新增/修改本人授课课程成绩 | 仅查看本人成绩 |

## 接口级数据范围

### 学生接口

- 管理员：可查看全部学生。
- 教师：可查看学生基础信息。
- 学生：只能查看本人学生档案。

### 课程接口

- 管理员：可查看全部课程并维护课程数据。
- 教师：只能查看本人授课课程。
- 学生：只能查看本人已有成绩记录关联的课程。

### 成绩接口

- 管理员：可查看和维护全部成绩。
- 教师：只能查看、录入、修改本人授课课程的成绩。
- 学生：只能查看本人成绩与本人成绩统计。

### 仪表盘接口

- 管理员：返回全量统计。
- 教师：只返回本人授课与所属院系相关统计。
- 学生：只返回本人及所属院系相关统计。

## 前后端配合原则

- 前端负责隐藏无权限菜单、按钮和筛选项。
- 后端必须继续做二次校验，不能依赖前端隐藏实现安全控制。
